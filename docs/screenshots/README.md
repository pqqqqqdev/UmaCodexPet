# Screenshot publication plan

The main README does not require screenshots. Add optional project screenshots
here only after they are cleared for publication; do not commit a generated pet
atlas merely to make the repository look complete.

| Suggested filename | Capture | Redaction checklist |
| --- | --- | --- |
| `01-mini-loaded.png` | UmaViewer showing one correctly loaded Mini model | Crop other desktop content; remove local paths |
| `02-selection-catalogs.png` | The F7-generated catalog folder and a comments-only override template | Remove username and install path; show no proprietary bundles |
| `03-export-complete.png` | UmaPetForge completion log beside `EXPORT_COMPLETE.txt` | Remove username, install path, and unrelated logs |
| `04-atlas-preview.png` | One atlas in a checkerboard transparency viewer | Confirm redistribution permission first |

After adding approved images, reference them from the relevant root README
section with relative Markdown links.

Do not commit game files, asset bundles, databases, extracted 3D models,
textures, or animations. A screenshot can still contain copyrighted material;
its small size or diagnostic purpose does not automatically make publication
permitted.
